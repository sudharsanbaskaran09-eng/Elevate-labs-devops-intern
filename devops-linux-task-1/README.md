# DevOps Internship – Task 1  
## Linux Environment Setup & Exploration

### Overview
This repository contains the work completed for **Task 1** of my DevOps Internship at **Elevate Labs**.  
The task focuses on setting up a Linux environment and exploring essential Linux commands used in daily DevOps workflows.

---

### Tools & Environment
- Windows Subsystem for Linux (WSL)
- Ubuntu 22.04 LTS
- Windows Terminal

---

### Tasks Performed
- Installed and configured Ubuntu using WSL
- Explored Linux directory structure using `pwd`, `ls`, and `cd`
- Created and deleted directories and files using `mkdir`, `touch`, `rm`, and `rmdir`
- Edited and viewed files using `nano`, `cat`, and `less`
- Understood and modified file permissions using `ls -l` and `chmod`
- Monitored system resources using `top`, `df -h`, and `free -m`

---

### Key Learnings
- Difference between root user and normal user
- Linux file permissions and ownership
- Safe file and directory management
- Basic system monitoring in Linux

---

### Outcome
This task helped me gain hands-on experience with Linux system navigation and administration, building a strong foundation for further DevOps learning and real-world system management.
